Preguntas de Laboratorio #1: HTML

1. ¿Cuál es la diferencia entre internet y la World Wide Web?

El internet es una extensa red de computadoras conectadas entre sí alrededor del mundo. En cambio, la World Wide Web es un sistema que, a través de internet, nos permite navegar entre páginas con hipervínculos.

2. ¿Cuáles son las partes de un URL?

  • Protocolo HTTPS: HyperText Transfer Protocol. Transfiere información entre actores dentro de la WWW. La 'S' hace referencia a la seguridad, pues cifra la información para prevenir daños en caso ser interceptada.
  • Subdominio: Se encuentra entre el protocolo y el dominio.
  • Dominio: Nombre único que identifica a una página web.
  • TLD: Top Level Domain. Se encuentra al final del dominio.
  • Ruta: Páginas y subpáginas que pueden encontrarse en el sitio
  • Parámetro: Después del signo '?' y separado con '&'. Puede referirse a búsqueda.

    • 3. ¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

  • GET: Solicita al servidor una información o representación de un recurso en un formato.
  • HEAD: Solicita sólo información de la cabecera.
  • POST: Para enviar datos al servidor. Ejemplo: Formularios
  • PUT: Similar a post, pero actualiza datos ya existentes en el servidor.
  • PATCH: Similar a Put, pero los datos se modifican de manera parcial.
  • DELETE: Elimina algún dato

    • 4. ¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?

    El método POST, pues queremos que los datos ingresados lleguen al servidor.

    5. ¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?

    Se utiliza el método get, pues solicita la información de la página.

    6. Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?

    No ocurrió error. Significa que la solicitud ha sido exitosa.

    7. ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?

    Sí es responsabilidad del desarrollador, pues significa que el servidor no encontró el recurso

    8. ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?

    Sí es su responsabilidad, pues significa que existe un error interno.

    9. ¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML 4 que en HTML5 estén desaprobados.

    Significa que ya no se utiliza, mas sigue funcionando en páginas que cuentan con él. Algunos son:

  • 'big': utilizado para hacer más grande el texto
  • 'center': para centrar texto
  • 'frameset': para dividir la ventana en distintas secciones

    • 10. ¿Cuáles son las diferencias principales entre HTML 4 y HTML5?

    HTML 5 es una mejora de algunos aspectos de HTML 4.

    11. ¿Qué componentes de estructura y estilo tiene una tabla?

    Estructura:
  • table: crea la tabla
  • tr: fila de la tabla
  • td: información de la celda
    •  Estilo:
  • border: para el borde de la tabla
  • width: para el ancho de la tabla
  • height: para el alto de la tabla

    • 12. ¿Cuáles son los principales controles de una forma (formulario) HTML5?

  • input: para ingresar texto
  • textarea: para ingresar texto más largo
  • select: para seleccionar una opción de un menú
  • button: para enviar el formulario

    • 13. ¿Qué tanto soporte HTML5 tiene el navegador que utilizas?

    Google Chrome tiene un gran soporte de HTML5

    14. ¿Cuál es el ciclo de vida de los sistemas de información?

    Abarca a identificación de una necesidad, planificación e ideas, desarrollo y mantenimiento. Termina cuando el programa es obsoleto o es reutilizado para generar uno nuevo.

    15. ¿Cuál es el ciclo de desarrollo de sistemas de información?

    Abarca sólo las etapas de creación (desarrollo) y mantenimiento del programa.

    Preguntas de Laboratorio #3: CSS

    1. Como ingeniero de software, ¿cuál es tu recomendación sobre el uso de !important en un CSS?

    !important establece prioridad sobre otros estilos establecidos. Recomendaría ser cuidadosos con su uso, teninendo en cuenta su impacto en el código y manteniendo un orden en la jerarquía de estilos.

    2. Si se pone una imagen de fondo en una página HTML, ¿por qué debe escogerse con cuidado?

    Porque puede dificultar la lectura del texto y, por ende, interferir en la experiencia del usuario. Además, debe asegurarse de que el tamaño de la imagen sea compatible con el de la pantalla.

    3. Como ingeniero de software, ¿cuál es tu recomendación al elegir las unidades de un propiedad de estilo entre %, px y pt?

  • %: relativo al tamaño del contenedor
  • px: son pixeles, así que es una medida fija
  • pt: relativo al tamaño de la pantalla

    • Recomendaría utilizar % por ser una medida relativa ajustable a cualquier pantalla. 'px' y 'pt' son medidas absolutas y, por ello, deben ajustarse a un espacio específico.

    4. ¿Por qué el uso de una versión minimizada del CSS mejora el rendimiento del sitio?

    Porque reduce el tamaño del archivo al eliminar los identados y espacios. Esto disminuye el tiempo de carga del archivo.
